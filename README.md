# Telegram-бот для отслеживания акций MOEX

Личный Telegram-бот проверяет российские акции из `tickers.txt` через MOEX ISS API. Бот берёт две последние закрытые свечи выбранного таймфрейма и сравнивает:

```text
close последней закрытой свечи > close предыдущей закрытой свечи
```

Незакрытая текущая свеча не используется.

## Таймфреймы

Ручная проверка работает для всех таймфреймов:

- `1m` — 1 минута, MOEX ISS `interval=1`;
- `10m` — 10 минут, MOEX ISS `interval=10`;
- `1h` — 1 час, MOEX ISS `interval=60`;
- `1d` — 1 день, MOEX ISS `interval=24`;
- `1w` — 1 неделя, MOEX ISS `interval=7`;
- `1mo` — 1 месяц, MOEX ISS `interval=31`.

Свечи `1m` и `10m` берутся напрямую из MOEX ISS. Ручной агрегации коротких свечей в коде нет.

## Что показывает бот

В отчётах больше не показываются объёмы. Вместо объёма бот показывает оборот в рублях из поля MOEX `value`, если это поле пришло в ответе API.

Примеры формата оборота:

- `523000` → `523 тыс ₽`;
- `5520000` → `5.52 млн ₽`;
- `5520000000` → `5.52 млрд ₽`.

Если MOEX не вернул `value`, бот не падает, не показывает строку оборота и пишет это в `bot.log`.

## Автоуведомления

По умолчанию автоматические уведомления включены только для:

- дневного отчёта;
- недельного отчёта;
- месячного отчёта.

Таймфреймы `1m`, `10m` и `1h` используются только для ручной проверки через кнопку `Проверить сейчас`, чтобы бот не спамил короткими уведомлениями.

Автоотчёты защищены от дублей через поля в `user_settings.json`:

- `last_sent_daily_candle`;
- `last_sent_weekly_candle`;
- `last_sent_monthly_candle`.

Если в день проверки торгов не было и новая закрытая свеча не появилась, бот ничего не отправляет и пишет запись в лог.

## Создание Telegram-бота

1. Откройте Telegram и найдите `@BotFather`.
2. Отправьте `/newbot`.
3. Введите имя бота.
4. Введите username, который заканчивается на `bot`.
5. Скопируйте токен в `.env` как `TELEGRAM_BOT_TOKEN`.

Не публикуйте `.env` и не отправляйте токен посторонним.

## Настройка `.env`

Создайте `.env` из примера:

```powershell
copy .env.example .env
notepad .env
```

Пример:

```dotenv
TELEGRAM_BOT_TOKEN=ваш_токен_от_BotFather
TELEGRAM_CHAT_ID=
ALLOWED_USER_ID=
TIMEZONE=Europe/Moscow
MARKET=MOEX
DEFAULT_TIMEFRAME=1d
AUTO_DAILY_REPORT=true
AUTO_WEEKLY_REPORT=true
AUTO_MONTHLY_REPORT=true
DAILY_REPORT_TIME=23:55
WEEKLY_REPORT_DAY=FRIDAY
WEEKLY_REPORT_TIME=23:55
MONTHLY_REPORT_TIME=23:55
USER_SETTINGS_PATH=user_settings.json
```

`TELEGRAM_CHAT_ID` можно оставить пустым. После `/start` бот сохранит `chat_id` в `user_settings.json`.

Чтобы ограничить доступ одним пользователем, укажите:

```dotenv
ALLOWED_USER_ID=123456789
```

## Тикеры

Заполните `tickers.txt`, по одному тикеру на строку:

```text
SBER
GAZP
LKOH
YDEX
```

Пустые строки и комментарии после `#` игнорируются.

## Кнопки

- `▶️ Старт` — текущие настройки и источник данных.
- `🔄 Рестарт` — сброс настроек без удаления истории Telegram.
- `🔍 Проверить сейчас` — ручной отчёт по выбранному таймфрейму.
- `⏱ Таймфрейм` — выбор `1m`, `10m`, `1h`, `1d`, `1w`, `1mo`.
- `🔔 Уведомления` — отдельные переключатели дневного, недельного и месячного отчёта.
- `📄 Последний отчёт` — последний сохранённый ручной отчёт.
- `💰 Оборот` — топ-10 тикеров по обороту последней закрытой свечи.
- `📋 Мои тикеры` — список из `tickers.txt`.
- `⚙️ Настройки` — таймфрейм, расписание отчётов, статусы уведомлений и timezone.
- `❓ Помощь` — краткая справка.

## Ручная проверка

Нажмите `⏱ Таймфрейм`, выберите период и нажмите `Проверить сейчас`.

Бот:

1. получает свечи из MOEX ISS;
2. сортирует их по `begin/end`;
3. выбирает две последние закрытые свечи;
4. сравнивает `close_last > close_previous`;
5. показывает тикеры, которые прошли условие;
6. добавляет период последней и предыдущей свечи;
7. показывает оборот в рублях, если MOEX вернул `value`.

## Автоматические отчёты

Дневной отчёт запускается каждый день в `DAILY_REPORT_TIME`.

Недельный отчёт запускается в день `WEEKLY_REPORT_DAY` и время `WEEKLY_REPORT_TIME`.

Месячный отчёт проверяется каждый день в `MONTHLY_REPORT_TIME`; сообщение отправляется только когда появилась новая закрытая месячная свеча.

## Запуск на Windows

Откройте PowerShell в папке проекта:

```powershell
cd C:\Users\admin\Desktop\test-codex
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
python bot.py
```

Если PowerShell запрещает активацию:

```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
.\.venv\Scripts\Activate.ps1
```

## Остановка

Если бот запущен в текущем окне PowerShell:

```text
Ctrl+C
```

Если бот запущен отдельным процессом:

```powershell
Get-Process python
Stop-Process -Id <PID>
```
