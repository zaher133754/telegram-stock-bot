# Telegram-бот для отслеживания акций MOEX

Личный Telegram-бот проверяет российские акции из `tickers.txt` через MOEX ISS API и ищет тикеры по условию:

```text
close последней закрытой свечи > high предыдущей закрытой свечи
```

Простыми словами: акция закрылась выше максимума предыдущей свечи.

Бот использует только закрытые свечи. Текущая незакрытая свеча не используется.

## Таймфреймы

Доступны ручная проверка и автоуведомления для всех таймфреймов:

- `1m` — 1 минута, MOEX ISS `interval=1`;
- `10m` — 10 минут, MOEX ISS `interval=10`;
- `1h` — 1 час, MOEX ISS `interval=60`;
- `1d` — 1 день, MOEX ISS `interval=24`;
- `1w` — 1 неделя, MOEX ISS `interval=7`;
- `1mo` — 1 месяц, MOEX ISS `interval=31`.

Свечи `1m` и `10m` берутся напрямую из MOEX ISS. Ручной агрегации коротких свечей в коде нет.

## Ручная проверка

Кнопка `🔍 Проверить сейчас` работает для всех таймфреймов: `1m`, `10m`, `1h`, `1d`, `1w`, `1mo`.

Бот:

1. получает свечи из MOEX ISS;
2. сортирует их по `begin/end`;
3. выбирает две последние закрытые свечи;
4. сравнивает `close_last > high_previous`;
5. показывает тикеры, которые прошли условие;
6. добавляет период последней и предыдущей свечи;
7. показывает оборот в рублях, если MOEX вернул `value`.

Статусы `X2`, `X3`, `X4` в ручной проверке не показываются.

## Автоуведомления

Автоуведомления работают сразу по нескольким таймфреймам. По умолчанию включены базовые уведомления:

- `1d` — 1 день;
- `1w` — 1 неделя;
- `1mo` — 1 месяц.

В разделе `🔔 Уведомления` пользователь может дополнительно включить короткие таймфреймы `1m`, `10m` и `1h`. Например, если включить `10m`, бот будет отправлять уведомления по закрытию 10-минутной свечи, а базовые уведомления за 1 день, 1 неделю и 1 месяц останутся включёнными.

Для `10m` бот проверяет появление новой закрытой 10-минутной свечи и отправляет отчёт, если найдёт тикеры, у которых:

```text
close последней закрытой 10-минутной свечи > high предыдущей закрытой 10-минутной свечи
```

Если дополнительно включён `1h`, отчёты отправляются после закрытия новой часовой свечи. Для `1d`, `1w` и `1mo` логика такая же: максимум один автоотчёт на одну новую закрытую свечу каждого включённого таймфрейма.

Планировщик проверяет новые закрытые свечи примерно раз в минуту. Если новой свечи нет, бот ничего не отправляет. Если подходящих тикеров нет, пустой автоотчёт по умолчанию не отправляется, запись остаётся в логе.

## Статусы X2/X3

Статусы `X2`, `X3`, `X4` показываются только в автоуведомлениях.

- `X2` означает, что тикер второй раз подряд попал в автоотчёт на выбранном таймфрейме.
- `X3` означает, что тикер третий раз подряд попал в автоотчёт.
- Если тикер перестал попадать в очередной автоотчёт, его счётчик по этому таймфрейму сбрасывается.

Счётчики считаются отдельно для каждого таймфрейма. Например, `SBER` может иметь `X3` на `10m` и не иметь статуса на `1h`.

## Оборот

В отчётах не показывается `volume`. Вместо объёма бот показывает оборот в рублях из поля MOEX `value`, если это поле пришло в ответе API.

Примеры формата оборота:

- `523000` → `523 тыс ₽`;
- `5520000` → `5.52 млн ₽`;
- `5520000000` → `5.52 млрд ₽`.

Если MOEX не вернул `value`, бот не падает, не показывает строку оборота и пишет предупреждение в `bot.log`.

## Настройки пользователя

В `user_settings.json` для пользователя хранятся:

- `chat_id`;
- `user_id`;
- `selected_timeframe` — таймфрейм ручной проверки;
- `notification_timeframes` — список таймфреймов автоуведомлений;
- `auto_notifications_enabled` — включены ли уведомления;
- `last_sent_candle_times` — последние обработанные свечи по таймфреймам;
- `streaks` — счётчики повторного попадания тикеров по таймфреймам.

Кнопка `🔄 Рестарт` не перезапускает программу и не удаляет историю Telegram. Она сбрасывает настройки пользователя к значениям:

```text
selected_timeframe = 1d
notification_timeframes = 1d, 1w, 1mo
auto_notifications_enabled = true
last_sent_candle_times = пусто
streaks = пусто
```

## Создание Telegram-бота

1. Откройте Telegram и найдите `@BotFather`.
2. Отправьте `/newbot`.
3. Введите имя бота.
4. Введите username, который заканчивается на `bot`.
5. Скопируйте токен в `.env` как `TELEGRAM_BOT_TOKEN`.

Не публикуйте `.env` и не отправляйте токен посторонним.

## Настройка `.env`

Создайте `.env` из примера:

```powershell
copy .env.example .env
notepad .env
```

Пример:

```dotenv
TELEGRAM_BOT_TOKEN=ваш_токен_от_BotFather
TELEGRAM_CHAT_ID=
ALLOWED_USER_ID=
TIMEZONE=Europe/Moscow
MARKET=MOEX
DEFAULT_TIMEFRAME=1d
DEFAULT_NOTIFICATION_TIMEFRAMES=1d,1w,1mo
AUTO_NOTIFICATIONS=true
USER_SETTINGS_PATH=user_settings.json

TICKERS_FILE=tickers.txt
LOG_FILE=bot.log

MOEX_BOARD=TQBR
MOEX_TIMEOUT_SECONDS=10
SCHEDULER_INTERVAL_SECONDS=60
```

`TELEGRAM_CHAT_ID` можно оставить пустым. После `/start` бот сохранит `chat_id` в `user_settings.json`.

Чтобы ограничить доступ одним пользователем, укажите:

```dotenv
ALLOWED_USER_ID=123456789
```

Старые переменные отдельных daily/weekly/monthly-уведомлений больше не нужны. Новая логика использует список `notification_timeframes`. Базовые `1d`, `1w` и `1mo` добавляются автоматически.

## Тикеры

Заполните `tickers.txt`, по одному тикеру на строку:

```text
SBER
GAZP
LKOH
YDEX
```

Пустые строки и комментарии после `#` игнорируются.

## Кнопки

- `▶️ Старт` — текущие настройки и источник данных.
- `🔄 Рестарт` — сброс настроек без удаления истории Telegram.
- `🔍 Проверить сейчас` — ручной отчёт по выбранному таймфрейму.
- `⏱ Таймфрейм` — выбор таймфрейма ручной проверки.
- `🔔 Уведомления` — базовые `1d/1w/1mo`, выбор дополнительных коротких таймфреймов и общий переключатель.
- `📄 Последний отчёт` — последний сохранённый ручной отчёт.
- `💰 Оборот` — топ-10 тикеров по обороту последней закрытой свечи.
- `📋 Мои тикеры` — список из `tickers.txt`.
- `⚙️ Настройки` — таймфреймы, статус уведомлений, источник данных, timezone и логика отбора.
- `❓ Помощь` — краткая справка.

## Запуск на Windows

Откройте PowerShell в папке проекта:

```powershell
cd C:\Users\admin\Desktop\test-codex
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
python bot.py
```

Если PowerShell запрещает активацию:

```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
.\.venv\Scripts\Activate.ps1
```

## Остановка

Если бот запущен в текущем окне PowerShell:

```text
Ctrl+C
```

Если бот запущен отдельным процессом:

```powershell
Get-Process python
Stop-Process -Id <PID>
```

## Логи

Логи пишутся в:

```text
bot.log
```

Посмотреть последние строки:

```powershell
Get-Content .\bot.log -Tail 50
```
